  <action>
    Install dependencies: `uv add asyncpraw feedparser pytrends`

    Create `src/pmtb/research/agents/__init__.py` with exports for all 4 agent classes.

    Create `src/pmtb/research/agents/reddit.py` — `RedditAgent`:
    - `__init__(self, classifier: SentimentClassifier, client_id: str | None, client_secret: str | None, user_agent: str, results_limit: int = 10)`
    - `source_name = "reddit"`
    - Category-to-subreddit mapping: {"politics": ["politics", "PoliticalDiscussion"], "economics": ["economics", "finance"], "finance": ["wallstreetbets", "stocks", "investing"], "sports": ["sports", "sportsbook"], "weather": ["weather"], "science": ["science"], "general": ["news", "worldnews"]}
    - `async def fetch(self, candidate, query) -> SourceResult`:
      1. If no client_id, return empty SourceResult with source="reddit"
      2. Get subreddits for candidate.category (fallback to "general")
      3. `async with asyncpraw.Reddit(...)` context manager
      4. For each mapped subreddit: fetch hot posts (limit=results_limit)
      5. Also run `reddit.subreddit("all").search(query, limit=results_limit)`
      6. Classify each post title with SentimentClassifier
      7. Return SourceResult with all classifications and raw_data={post titles, subreddits searched}
    - Log with loguru `.bind(source="reddit", ticker=candidate.ticker)`

    Create `src/pmtb/research/agents/rss.py` — `RSSAgent`:
    - `__init__(self, classifier: SentimentClassifier, feeds_by_category: dict[str, list[str]], results_limit: int = 10)`
    - `source_name = "rss"`
    - `async def fetch(self, candidate, query) -> SourceResult`:
      1. Get feed URLs for candidate.category (fallback to "general")
      2. If no feeds, return empty SourceResult
      3. For each feed URL: fetch with httpx.AsyncClient, parse with feedparser.parse(response.text) — NEVER pass URL to feedparser (blocks event loop)
      4. Filter entries by query keyword relevance (simple substring match on title)
      5. Classify each relevant entry's title+summary with SentimentClassifier
      6. Return SourceResult with classifications and raw_data={feed_urls, entry_count}
    - Use tenacity retry for httpx fetch (same pattern as kalshi_retry)

    Create `src/pmtb/research/agents/trends.py` — `TrendsAgent`:
    - `__init__(self, classifier: SentimentClassifier)`
    - `source_name = "trends"`
    - `async def fetch(self, candidate, query) -> SourceResult`:
      1. Build pytrends payload with query keyword
      2. Call `interest_over_time()` via `asyncio.to_thread()` — pytrends is synchronous
      3. If DataFrame is empty or 429 error, return empty SourceResult
      4. Derive momentum: compare last 7 days average to previous 7 days. Rising = bullish signal, falling = bearish.
      5. Also call `related_queries()` via `asyncio.to_thread()`, store in raw_data
      6. Return SourceResult with single SignalClassification (momentum-based) and raw_data={interest_data, related_queries}
    - Wrap pytrends calls with tenacity retry (exponential backoff for 429s)
    - Add 1-second sleep between pytrends requests using asyncio.sleep

    Create `tests/research/test_agents.py`:

    - Mock httpx responses for RSSAgent tests
    - Mock pytrends TrendReq for TrendsAgent tests
    - Test each agent returns SourceResult
    - Test each agent passes isinstance(agent, ResearchAgent)
    - Test RedditAgent with no credentials returns empty SourceResult
    - Test RSSAgent with no feeds returns empty SourceResult
    - Test TrendsAgent with empty DataFrame returns empty SourceResult
