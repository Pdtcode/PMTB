<objective>
Build the LossClassifier that diagnoses losing trades using rule-based heuristics with Claude fallback for ambiguous cases. Each losing trade gets classified by one of 6 error types (edge_decay, signal_error, llm_error, sizing_error, market_shock, unknown) with the classification persisted to the LossAnalysis DB table.

Purpose: Understanding WHY trades lose is essential for the learning loop — it tells the retraining system which signals to weight differently and whether model degradation is systematic or random.

Output: `src/pmtb/performance/loss_classifier.py` with full test coverage.
</objective>

Heuristic: resolved_outcome disagrees with trade side AND p_market at prediction time was closer to resolution than p_model
    - signal_error: majority of research signals for the trade's market were wrong direction. Heuristic: count bullish vs bearish signals for market_id; if majority sentiment aligned with trade side but outcome was opposite, it's signal_error
    - llm_error: Claude was used (ModelOutput.used_llm=True) AND the XGBoost-only p_model (approximated from signal_weights) would have been on the correct side. Heuristic: used_llm=True AND removing Claude's contribution would have changed the decision
    - sizing_error: the edge was correct (p_model was on the right side of the outcome) but loss was due to position being too large relative to edge magnitude. Heuristic: resolved_outcome matches predicted direction but pnl < 0 (e.g., correct direction but overleveraged on a narrow edge that decayed)
    - market_shock: rapid large price movement inconsistent with all signals — neither model nor signals predicted the outcome. Heuristic: all signals were neutral or weakly directional AND p_model was near 0.5 (within 0.1)
    - unknown: none of the above rules match — queue for Claude analysis
    - Claude fallback: when classified_by="rules" returns unknown AND anthropic_api_key is set, call Claude with trade context for classification. Use lazy AsyncAnthropic import pattern (same as SentimentClassifier)
    - classify_trade returns LossAnalysisResult
    - classify_and_persist writes LossAnalysis row to DB
    - Only classifies losing trades (pnl < 0)
    - Prometheus counter: LOSS_CLASSIFICATIONS with error_type and classified_by labels
