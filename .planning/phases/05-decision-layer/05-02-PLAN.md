<objective>
Build the PositionTracker (in-memory async dict synced from DB) and the RiskManager (stateful gate enforcing portfolio limits, drawdown halt, VaR, duplicate detection, and auto-hedge). Together these form the third and final gate in the Edge -> Size -> Risk pipeline.

Purpose: The risk manager is the last line of defense before a trade reaches the executor. It enforces hard portfolio limits, detects drawdown breaches, prevents duplicate bets, and triggers hedges on reversed positions. The position tracker provides the fast in-memory state that risk checks query.

Output: `src/pmtb/decision/tracker.py` and `src/pmtb/decision/risk.py` with full test coverage.
</objective>

    - test_auto_hedge_no_trigger: Edge=-0.01 (above -0.03) -> no hedge triggered

  <action>
    Create `src/pmtb/decision/risk.py` with `RiskManager` class:

    - `__init__(self, tracker: PositionTracker, session_factory: async_sessionmaker, max_exposure: float, max_single_bet: float, var_limit: float, max_drawdown: float, hedge_shift_threshold: float, portfolio_value: float)`:
      - Store all parameters. portfolio_value is the current total portfolio value in dollars (passed from pipeline).

    - `async def check(self, decision: TradeDecision) -> TradeDecision`:
      - Sequential checks (short-circuit on first rejection):
        1. Check halt flag in TradingState table -> DRAWDOWN_HALTED
        2. Check duplicate: tracker.has_position(decision.ticker) -> DUPLICATE_POSITION
        3. Check drawdown: read peak_portfolio_value from TradingState, compute drawdown = (peak - current) / peak, if >= max_drawdown -> DRAWDOWN_HALTED. Update peak if current > peak.
        4. Check max single bet: decision.quantity * decision.p_market > max_single_bet * portfolio_value -> MAX_SINGLE_BET
        5. Check max exposure: tracker.total_exposure() + trade_cost > max_exposure * portfolio_value -> MAX_EXPOSURE
        6. Check VaR: compute portfolio VaR with current positions, simulate adding new position, if VaR exceeds var_limit * portfolio_value -> VAR_EXCEEDED
      - If all pass: return decision unchanged (still approved)
      - Use model_copy(update={approved: False, rejection_reason: ...}) for rejections

    - `async def check_hedge(self, prediction: PredictionResult, candidate: MarketCandidate) -> TradeDecision | None`:
      - If not tracker.has_position(prediction.ticker): return None
      - edge = prediction.p_model - candidate.implied_probability
      - If edge < -hedge_shift_threshold: return TradeDecision(approved=True, side="sell", ticker, cycle_id, edge, p_model, p_market)
      - Else: return None

    - `def _compute_var(self, position_values: list[float]) -> float`:
      - If len < 2: return 0.0
      - mu = statistics.mean(values), sigma = statistics.stdev(values)
      - return mu - 1.645 * sigma

    Add Prometheus metrics:
    - `RISK_REJECTIONS = Counter("pmtb_risk_rejections_total", "...", ["reason"])`
    - Increment on each rejection with the reason label

    For tests: Mock the PositionTracker (inject mock with controlled return values). Mock session_factory for TradingState queries. Test each check independently. Use concrete values that clearly cross or stay within thresholds.
