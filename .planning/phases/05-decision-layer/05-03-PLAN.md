  <action>
    Create `src/pmtb/decision/watchdog.py`:

    CRITICAL DESIGN CONSTRAINTS (from CONTEXT.md locked decisions):
    - daemon=False on multiprocessing.Process — must survive main process crash
    - No shared DB connections across process boundary — create engine INSIDE run_watchdog
    - Communicate halt signal via TradingState DB flag only
    - Poll every 30 seconds
    - On halt: set DB flag AND cancel pending orders via Kalshi API
    - Must be simple and reliable, not clever

    Implementation:
    - `POLL_INTERVAL_SECONDS = 30`
    - `async def _check_and_act(session_factory, settings) -> None`:
      1. Query all open positions: sum(quantity * avg_price) for current portfolio value
      2. Read peak_portfolio_value from TradingState (default to current if not set)
      3. If current > peak: update peak in TradingState
      4. Compute drawdown = (peak - current) / peak
      5. If drawdown >= settings.max_drawdown:
         a. Set TradingState key="trading_halted" value="true"
         b. Log critical warning with loguru
         c. Cancel pending orders: query Order table for status="pending", call Kalshi API to cancel each (best effort — log errors, don't crash)
         d. Increment WATCHDOG_HALT_TRIGGERS Prometheus counter
    - `async def _watchdog_loop(settings) -> None`:
      - Create engine and session_factory from settings.database_url (own connection pool)
      - Initialize Kalshi client from settings (own client instance)
      - while True: try _check_and_act, except log error, await asyncio.sleep(POLL_INTERVAL_SECONDS)
      - Watchdog must NEVER crash — catch all exceptions, log, continue
    - `def run_watchdog(settings_dict: dict) -> None`:
      - Entry point for multiprocessing.Process target
      - Reconstruct Settings from dict (can't share Pydantic objects across fork)
      - asyncio.run(_watchdog_loop(settings))
    - `def launch_watchdog(settings: Settings) -> multiprocessing.Process`:
      - Serialize settings to dict via settings.model_dump()
      - Create Process(target=run_watchdog, args=(settings_dict,), daemon=False)
      - proc.start()
      - return proc

    For tests: Test the _check_and_act function directly (not the process spawn). Mock session_factory and Kalshi client. Inject controlled portfolio values and verify halt flag is set. For the process test, use a simple smoke test that verifies the process starts and can be terminated (don't test full DB polling in unit tests — that's integration).

    Add Prometheus metrics:
    - WATCHDOG_HALT_TRIGGERS Counter
    - WATCHDOG_POLLS Counter (increment each cycle)

      - Store all injected components

    - `async def evaluate(self, predictions: list[PredictionResult], candidates: list[MarketCandidate]) -> list[TradeDecision]`:
      - Build a dict of candidates keyed by ticker for O(1) lookup
      - results: list[TradeDecision] = []

        1. If prediction.is_shadow: append rejected TradeDecision(approved=False, rejection_reason=SHADOW) and continue

        6. Risk gate: await risk_manager.check(decision). If not approved: log rejection, increment counter, append, continue
        7. If still approved: log approval, increment DECISION_APPROVALS counter, append
      - Return results
