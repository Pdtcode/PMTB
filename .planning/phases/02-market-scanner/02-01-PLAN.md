<task type="auto" tdd="true">
  <name>Task 2: Filter functions with rejection counting and volatility tracker</name>
  <files>src/pmtb/scanner/filters.py, tests/scanner/test_filters.py</files>
  <behavior>
    - filter_liquidity: markets with float(open_interest_fp) < threshold are rejected; returns (passing_list, rejected_count)
    - filter_volume: markets with float(volume_24h_fp) < threshold are rejected
    - filter_spread: markets with (float(yes_ask_dollars) - float(yes_bid_dollars)) > max_spread are rejected; markets missing bid/ask fields are rejected
    - filter_ttr: markets resolving in < min_ttr_hours OR > max_ttr_days from now are rejected; close_time parsed from ISO 8601 string
    - filter_volatility: uses VolatilityTracker; markets with stdev < min_volatility are rejected; markets in warmup (< N snapshots) PASS the filter (not rejected)
    - VolatilityTracker.record_and_get(ticker, price) returns None during warmup, returns stdev after warmup
    - VolatilityTracker uses collections.deque(maxlen=50) per ticker
    - All filters are pure functions taking (markets: list[dict], threshold_value) except volatility which also takes the tracker
  </behavior>
  <action>
    1. Create `src/pmtb/scanner/filters.py` with:
       - `filter_liquidity(markets: list[dict], min_open_interest: float) -> tuple[list[dict], int]` — filters on `float(m.get("open_interest_fp", "0"))`. DO NOT use `liquidity_dollars` (always returns 0, deprecated).
       - `filter_volume(markets: list[dict], min_volume_24h: float) -> tuple[list[dict], int]` — filters on `float(m.get("volume_24h_fp", "0"))`.
       - `filter_spread(markets: list[dict], max_spread: float) -> tuple[list[dict], int]` — computes `float(yes_ask_dollars) - float(yes_bid_dollars)`, rejects if > max_spread or if fields missing/zero.
       - `filter_ttr(markets: list[dict], min_hours: float, max_days: float) -> tuple[list[dict], int]` — parses `close_time` ISO 8601 string, computes timedelta from `datetime.now(UTC)`, rejects outside [min_hours, max_days] window.
       - `VolatilityTracker` class: dict of `deque(maxlen=50)` per ticker. `record_and_get(ticker: str, price: float) -> float | None` appends price, returns `statistics.stdev(history)` if `len >= warmup`, else None. Guard against len < 2 even if warmup is set low.
       - `filter_volatility(markets: list[dict], min_volatility: float, tracker: VolatilityTracker, warmup: int) -> tuple[list[dict], int]` — records `float(yes_bid_dollars)` as price snapshot. If tracker returns None (warmup), market PASSES. If stdev < min_volatility, market rejected.
       - Helper: `parse_close_time(raw: str) -> datetime` — handles ISO 8601 with "Z" suffix. Centralized parser.

    2. Create `tests/scanner/test_filters.py` with tests covering:
       - test_filter_liquidity: mix of passing/failing markets, correct rejection count
       - test_filter_volume: same pattern
       - test_filter_spread: passing, failing, missing fields
       - test_filter_ttr_too_soon: market resolving in 30 minutes excluded
       - test_filter_ttr_too_far: market resolving in 60 days excluded
       - test_filter_ttr_passing: market resolving in 5 days passes
       - test_volatility_warmup_skip: market with < warmup snapshots passes filter
       - test_filter_volatility: market with sufficient snapshots and low stdev rejected
       - test_filter_volatility_passing: market with sufficient stdev passes
       - test_volatility_tracker_deque_maxlen: verify deque does not grow beyond 50

    All filter tests use raw dict fixtures mimicking Kalshi API response format (string values for price fields).
  </action>
