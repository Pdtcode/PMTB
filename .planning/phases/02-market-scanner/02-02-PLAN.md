  <action>
    1. Create `src/pmtb/scanner/scanner.py` with class `MarketScanner`:

       Constructor: `__init__(self, client: KalshiClient, settings: Settings, session_factory=None)`.
       Stores client, settings, creates VolatilityTracker instance.

       **`_fetch_all_markets(self) -> list[dict]`:**
       Cursor-based pagination loop using `self._client._request("GET", "/trade-api/v2/markets", params={"status": "active", "limit": 1000, "cursor": cursor})`. Loop until cursor is None/empty. Return flat list of all market dicts. Log total count.

       **`_upsert_markets(self, markets: list[dict]) -> None`:**
       Use `get_session(self._session_factory)`. For each market, use `sqlalchemy.dialects.postgresql.insert` with `on_conflict_do_update` on ticker. Parse `close_time` with `parse_close_time()`. Batch upsert. `await session.commit()`. IMPORTANT: Must commit explicitly -- AsyncSession does NOT auto-commit.

       **`_apply_filters(self, markets: list[dict]) -> tuple[list[dict], dict[str, int]]`:**
       Apply filters sequentially: liquidity -> volume -> spread -> ttr -> volatility.
       Pass threshold values from `self._settings`. Record rejection count per filter.
       Log per-market rejection reason at DEBUG level using `logger.bind(ticker=m["ticker"]).debug(...)`.
       Return (passing_markets, {"rejected_liquidity": N, ...}).

       **`_enrich(self, markets: list[dict]) -> list[MarketCandidate]`:**
       For each passing market, fetch orderbook (`/trade-api/v2/markets/{ticker}/orderbook?depth=3`) and event (`/trade-api/v2/events/{event_ticker}`). Use `asyncio.gather()` with `asyncio.Semaphore(settings.scanner_enrichment_concurrency)` to limit concurrency.
       Build MarketCandidate from enriched data:
       - yes_bid from orderbook: best yes_dollars bid price (float of first element)
       - yes_ask from orderbook: 1.0 - best no_dollars bid price
       - implied_probability: (yes_bid + yes_ask) / 2
       - spread: yes_ask - yes_bid
       - volume_24h: float(market["volume_24h_fp"])
       - volatility_score: from VolatilityTracker (may be None during warmup)
       - event_context: {"title": event["title"], "event_ticker": event["event_ticker"]}
       Sort candidates by `abs(candidate.implied_probability - 0.5)` ascending (closest to 50% first).

       **`async def run_cycle(self) -> ScanResult`:**
       Generate cycle_id via uuid4. Record start time. Call _fetch_all_markets, _upsert_markets, _apply_filters, _enrich. Build and return ScanResult with all metadata.
       Log summary: total markets, candidates found, per-filter rejections, duration.

       **`async def run_forever(self) -> None`:**
       Infinite loop: call run_cycle(), then `await asyncio.sleep(self._settings.scan_interval_seconds)`. Wrap run_cycle in try/except to log errors without crashing the loop.

    Key implementation notes:
    - KalshiClient.get_markets() strips the cursor. Use _request() directly for pagination.
    - Orderbook response has `orderbook_fp.yes_dollars` and `orderbook_fp.no_dollars`. yes_ask = 1 - float(no_dollars[0][0]).

    - If orderbook is empty (no bids), skip that candidate gracefully.
    - close_time "Z" suffix must be handled: replace "Z" with "+00:00" for fromisoformat().
