    ```
    class FillTracker:
        def __init__(self, ws_client, kalshi_client, order_repo, settings):
            self._ws = ws_client        # KalshiWSClient
            self._rest = kalshi_client   # KalshiClient (REST)
            self._repo = order_repo      # OrderRepository
            self._settings = settings    # Settings

        async def run(self, stop_event: asyncio.Event) -> None:
            # Run three concurrent loops via asyncio.gather:
            # 1. _ws_fill_loop — subscribe to fill channel, process events
            # 2. _stale_canceller_loop — check every 60s for pending orders past timeout
            # 3. _rest_polling_loop — poll REST every stale_order_timeout/2 seconds as safety net
    ```

    **_ws_fill_loop(stop_event):**
    - Define `async def on_message(msg)` callback that calls `_handle_fill_event(msg)` if `msg.get("type") == "fill"`
    - Create asyncio task running `self._ws.run(on_message, channels=["fill"], market_tickers=[])` (account-level subscription — empty ticker list)
    - Wait for stop_event, then cancel the WS task
    - Wrap entire loop in try/except to log errors without crashing

    **_handle_fill_event(msg):**
    - Extract `kalshi_order_id = msg.get("order_id")`, `fill_price = msg.get("yes_price", msg.get("fill_price"))`, `filled_qty = msg.get("count", 0)`
    - Look up order via `self._repo.get_by_kalshi_id(kalshi_order_id)` — if None, log warning and return
    - Compute `slippage_cents = fill_price - float(order.price)`
    - Log fill event with slippage via loguru `.bind(slippage_cents=slippage_cents)`
    - Call `self._repo.update_fill(order.id, fill_price, filled_qty, "filled" if filled_qty >= order.quantity else "partial")`
    - Increment Prometheus counter: `FILL_EVENTS_TOTAL.inc()`

    **_stale_canceller_loop(stop_event):**
    - Every 60 seconds (use `asyncio.wait_for(stop_event.wait(), timeout=60.0)` pattern):
    - Call `self._repo.get_stale_orders(self._settings.stale_order_timeout_seconds)`
    - For each stale order:
      - Check REST status first: `self._rest.get_orders()` filtered, or try cancel directly
      - Call `self._rest.cancel_order(order.kalshi_order_id)` wrapped in try/except (handle 404 = already filled gracefully)
      - Call `self._repo.cancel_order(order.id)`
      - Increment Prometheus counter: `STALE_CANCELLATIONS_TOTAL.inc()`

    **_rest_polling_loop(stop_event):**
    - Every `stale_order_timeout_seconds // 2` seconds:
    - Call `self._rest.get_orders(status="filled")` to get recently filled orders
    - For each filled order from REST: check if DB order still shows "pending" — if so, update via `self._repo.update_fill()`
    - This is a safety net for missed WS events

    Add Prometheus metrics at module level:
    - `FILL_EVENTS_TOTAL = Counter("pmtb_fill_events_total", "Fill events processed from WS or REST")`
    - `STALE_CANCELLATIONS_TOTAL = Counter("pmtb_stale_cancellations_total", "Stale orders cancelled")`
    - `FILL_SLIPPAGE_CENTS = Histogram("pmtb_fill_slippage_cents", "Slippage in cents per fill")`

    - All tests use `pytest.mark.asyncio` (asyncio_mode="auto" in pyproject.toml)
